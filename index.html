Under the hood

How Sketchbook works

No accounts. No servers. Your ideas stay yours.

01
Your ideas live in your browser

When you type something in Sketchbook, it saves automatically to your browser's built-in storage — called localStorage. Think of it like a notepad that lives inside your browser tab. Nothing is sent to any server. No account is required. Close the tab, reopen it tomorrow: your ideas are still there. The catch: they live in this browser, on this device. Clear your browser data and they're gone — which is why GitHub backup exists.

02
Auto-save fires every keystroke

There's no Save button. Every time you type, a 0.8-second timer resets. When you stop typing for 0.8 seconds, the current state of your idea — title, beats, stage — is written to browser storage. The "Saving… / Saved" indicator in the top bar reflects this. If you close the window mid-word, you lose at most one second of work.

03
Versions are snapshots, not overwrites

When you click Save as new version, your current beats are locked — frozen exactly as they are — and a fresh copy opens for editing. Version 1 doesn't change. Version 2 starts from where Version 1 ended. You can flip between them using the version pills in the editor. This means you can explore a completely different direction without losing what you had. Most writers end up with 2–4 versions before something clicks.

04
GitHub backup: your ideas as real files

If you connect a GitHub repository in Settings, every auto-save also commits your idea as a .md (markdown) file to that repo. GitHub is a platform developers use to store and version-control code — but it works just as well for text. Each idea becomes a file named something like saltwater_v2.md. You can see every change ever made to it, recover old versions, and access it from any device. This is proper version control — the same system used to build every major app in the world — applied to your creative work.

To set it up: create a free GitHub account, make a new private repository (e.g. my-ideas), generate a Personal Access Token with repo permissions, and paste both into Settings. Sketchbook does the rest silently.

05
Export is just a file download

The Share screen converts your idea into a .md file — plain text with light formatting markers (# for headings, - for bullets). This format opens in Final Draft, Celtx, Google Docs, Word, or any text editor. It's deliberately simple: no proprietary format, no lock-in. If Sketchbook disappeared tomorrow, every exported file would still be readable in Notepad.

06
Beat suggestions are AI-generated, opt-in only

The Get beat ideas button pulls from a curated bank of structural beat patterns — common story shapes used across film, theatre, and television. They're not written by AI on the fly; they're pre-written prompts designed to unblock, not to write the story for you. Click one to append it to your beats. Click Different approach to rotate to another set. Nothing is added without your action. You remain the author — the suggestions are a trampoline, not a script.

07
Privacy: what goes where

Without GitHub connected: your ideas never leave your device. With GitHub connected: your ideas are committed to your private repository on GitHub's servers. GitHub's privacy policy applies to that data — your repo is private by default, visible only to you. Sketchbook itself has no backend, no database, no analytics, and no account system. There is nothing to hack because there is nothing stored centrally.
